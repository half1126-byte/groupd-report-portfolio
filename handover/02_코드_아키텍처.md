## 2. app.py 구조 (핵심 — 6,429줄)

### ⚠️ 가장 중요한 패턴: "함수 재정의(Override)"

app.py에서 **같은 이름의 함수가 여러 번 정의**됩니다.
Python에서는 **마지막 정의가 유효**하므로, 맨 아래 정의가 실제 동작합니다.

```python
# 예: get_action_plan_for_report()가 3번 정의됨
def get_action_plan_for_report():   # 1차 (기본)
    ...
def get_action_plan_for_report():   # 2차 (개선)
    ...
def get_action_plan_for_report():   # 3차 (최종) ← 이게 실행됨
    ...
```

**주의**: 수정할 때 반드시 **마지막 정의**를 수정해야 합니다!

## 4. 실행계획(액션플랜) 파이프라인

이 부분이 가장 복잡합니다. 데이터 흐름:

```
정책 사전 (POLICY dicts)
    ↓
TEAM_PACKAGE_REGISTRY (팀별 패키지 구성)
    ↓
_render_team_package_cards() — PM이 카드 UI에서 체크박스로 선택
    ↓
st.session_state["{team}_{mode}_selections"] = ["key1", "key2", ...]
    ↓
PM이 "선택 완료" 클릭
    ↓
_confirm_team_package_selection() — 키 → 아이템 객체로 변환
    ↓
st.session_state.action_plan_items[team_key] = [items...]
    ↓
_normalize_product_items() — 필드 정규화 (title, detail, desc, price, count_label 등)
    ↓
get_action_plan_for_report() — 보고서용 최종 포맷
    ↓
HTML 템플릿의 summary.action_plan으로 렌더링
```

### 정책 사전 구조

```python
MARKETING_PM_POLICY = {
    "review_kakaomap": {
        "title": "카카오맵 리뷰",
        "price": 10000,                   # 단가
        "tasks": ["건당 대체 20건"],       # 실행 내용
        "desc": "카카오맵에 병원 방문자 리뷰를 등록하여 지도 검색 노출 강화",
    },
    # ... 40개+ 상품
}
```

### 가격 모델

- **1 블로그건 = 200,000원** (`_BLOG_UNIT_KRW = 200_000`)
- 마케팅: `price × 건당_대체 = 200,000` (항상 일관)
- 디자인 이월: `10만원 = 1 치환건`, `20만원 = 2 치환건`
- 이월 → 치환: `이월건수 × 0.5 = 치환 가능 건수`

## 5. HTML 보고서 구조 (html_generator.py)

### Jinja2 템플릿 구조

```html
[헤더] — 치과명, 날짜, 뱃지
[부서별 섹션] × 6 (조건부 렌더링)
    ├ 부서 헤더 (아이콘, 이름, 월 표시)
    ├ 메트릭 카드 (전월 회색 / 당월 파란색)
    ├ TOP5 수평 바 차트 (전월 vs 당월)
    └ 부서별 고유 섹션 (포스팅 리스트, 키워드 랭킹 등)
[종합 분석 & 실행 계획]
    ├ AI 핵심 인사이트 (sparkles 아이콘 카드)
    ├ 담당자 코멘트
    ├ 분석 섹션 (선택)
    ├ 진단 (선택)
    └ 액션플랜 카드 (부서 태그 + 유형 + 상품명 + 설명 + 건수)
[푸터]
```
