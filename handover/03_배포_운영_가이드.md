# 배포 및 운영 가이드

## 1. 로컬 개발 환경 세팅

### 필요 조건
- Python 3.10 이상
- pip (또는 conda)

### 설치 및 실행

```bash
# 1. 저장소 클론
git clone https://github.com/groupddesignteam-cyber/report_v1.0.git
cd report_v1.0

# 2. 가상환경 생성 (선택사항이지만 권장)
python -m venv venv
source venv/bin/activate        # macOS/Linux
# venv\Scripts\activate         # Windows

# 3. 의존성 설치
pip install -r requirements.txt

# 4. 실행
streamlit run app.py
```

브라우저에서 `http://localhost:8501` 접속

### API 키 설정 (AI 요약 기능용)

```bash
# .streamlit/secrets.toml 생성 (이미 있으면 수정)
ANTHROPIC_API_KEY = "sk-ant-your-key-here"
# 또는
OPENAI_API_KEY = "sk-your-key-here"
```

> API 키 없어도 앱 자체는 정상 동작합니다. AI 요약 버튼만 비활성화됩니다.

## 2. Streamlit Community Cloud 배포

### 현재 배포 방식

1. GitHub `main` 브랜치에 push
2. Streamlit Cloud가 자동으로 감지하여 재배포
3. 약 1-2분 내 반영

### 배포 URL 관리

- Streamlit Cloud 대시보드: https://share.streamlit.io/
- GitHub 계정: `groupddesignteam-cyber`
- 앱 URL: Streamlit Cloud 대시보드에서 확인 가능

### Secrets 설정 (프로덕션)

Streamlit Cloud 대시보드 → 해당 앱 → Settings → Secrets에서 설정:

```toml
ANTHROPIC_API_KEY = "sk-ant-..."
```

> ⚠️ `.streamlit/secrets.toml`은 git에 포함하지 않습니다.

## 3. 자주 하는 작업

### 새 상품 추가하기

마케팅팀 PM 제안 상품 추가 예시:

**app.py에서 `MARKETING_PM_POLICY` dict를 찾아 추가:**
```python
"new_product_key": {
    "title": "새 상품 이름",
    "price": 100000,           # 단가 (원)
    "tasks": ["건당 대체 2건"], # 건당 대체 = 200,000 / price
    "desc": "상품 설명 텍스트",
},
```

**`TEAM_PACKAGE_REGISTRY`에서 groups에 prefix 추가 (해당 시):**
```python
"groups": [
    ...,
    {"prefix": "new_product", "label": "새 카테고리"},
],
```

### 새 팀(부서) 추가하기

1. `src/processors/` 에 새 프로세서 파일 생성
2. `src/processors/__init__.py`에 import 추가
3. `src/utils.py`의 `FILE_PATTERNS`에 파일 패턴 추가
4. `app.py`에서 프로세서 호출 추가
5. `src/reporting/html_generator.py`에서 해당 부서 섹션 추가

### 보고서 디자인 수정하기

- **Streamlit UI CSS**: `src/ui/style.css`
- **HTML 보고서 CSS/레이아웃**: `src/reporting/html_generator.py` 내부 Jinja2 템플릿
- **보고서 디자인 토큰**: `html_generator.py` 상단 `<style>` 태그 내 CSS 변수

### 버전 올리기

```python
# app.py 상단
APP_VERSION = "v3.9.0"  # 여기만 수정
```

## 4. 트러블슈팅

### 파일 업로드가 안 될 때

- 파일명이 `FILE_PATTERNS`(src/utils.py)의 정규식에 매칭되는지 확인
- 각 팀 파일명 규칙이 바뀌면 정규식 수정 필요

### 보고서에 특정 팀 데이터가 안 나올 때

- 해당 팀 프로세서가 데이터를 정상 반환하는지 확인
- `filtered_results`에 해당 키가 있는지 확인
- 디자인팀: `clinic_name`이 데이터의 `치과명`과 매칭되는지 확인

### desc(상품 설명)가 보고서에 안 나올 때

`desc` 파이프라인 체크:
1. 정책 dict에 `desc` 필드 있는지
2. `_confirm_team_package_selection()`에서 `desc` 전달하는지
3. `_normalize_product_items()`에서 `desc` 보존하는지
4. `get_action_plan_for_report()`에서 `desc`를 `plan_text`로 사용하는지

### Streamlit Cloud 배포 후 에러

```bash
# 로컬에서 먼저 컴파일 테스트
python -m py_compile app.py
python -m py_compile src/reporting/html_generator.py
```

## 5. 코드 수정 시 주의사항

### 1. 함수 재정의 패턴

같은 함수가 여러 번 정의된 경우, **반드시 맨 마지막 정의를 수정**해야 합니다.
중간 정의를 수정하면 아무 효과 없습니다.

```bash
# 특정 함수의 모든 정의 위치 찾기
grep -n "^def get_action_plan_for_report" app.py
```

### 2. desc/count_label 파이프라인

새 필드를 아이템에 추가할 때, 파이프라인 4곳을 모두 수정해야 합니다:
1. 정책 dict (원본 데이터)
2. `_confirm_team_package_selection()` (아이템 생성)
3. `_normalize_product_items()` (필드 보존)
4. `get_action_plan_for_report()` (보고서 전달)

### 3. HTML 보고서는 인라인 CSS

`html_generator.py`의 HTML은 이메일/다운로드용이므로 **모든 CSS가 인라인**입니다.
외부 CSS 파일 참조 불가. 스타일 변경 시 인라인으로 작성해야 합니다.

## 7. 개발 팁

- **py_compile로 항상 확인**: 배포 전 `python -m py_compile app.py` 필수
- **git push = 즉시 배포**: main 브랜치 push 시 자동 배포되므로 주의
- **app.py가 거대함**: 6,400줄이므로 `grep -n`으로 위치 찾는 습관 필요
- **Streamlit 리런 모델**: 체크박스 클릭 등 모든 상호작용 시 전체 스크립트 재실행
- **session_state**: 리런 간 데이터 유지는 `st.session_state`로만 가능
